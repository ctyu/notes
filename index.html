A smart way to eslint and scss-lint for front end
立即使用
安装 felint

安装前请确保你的电脑已经安装好了 npm 和 gem

npm install -g felint

初始化 felint

进入到你项目的根目录，运行

felint init

felint 会从https://github.com/youzan/felint-config下载所需的默认的配置文件，包括Git Hook脚本、eslint 和 css lint 的配置文件等，然后放到当前项目合适的位置，并执行内置脚本安装依赖。

使用自己的 felint 配置

如果你不想使用我们默认的felint-config校验，你可以 fork 出来修改为自己的 felint-config（修改方法参考 felint-config 的 readme ），然后在项目中手动配置你自己的 felint-config 仓库地址，在你的项目根目录添加一个 .felintrc 文件，其文件内容格式为：

{

    "gitHookUrl": "your url"

}

然后重新执行一次felint init即可。

扩展默认 felint 配置

如果你仅仅想简单修改默认配置项，你可以在 .felintrc 文件中简单添加覆盖项目，其内容格式为：

{

    "eslintrc_es6": {your rules}

    "eslintrc_es5": {your rules}

}

然后重新执行一次felint use或felint init即可。

注意，执行felint use命令的前提必须为已执行felint init

felint 对es6的支持

felint init命令默认初始化的为es5的语法规则，如果你的项目使用es6语法，请使用如下命令切换：

如尚未执行felint init：

felint init -6

如已执行felint init：

felint use -6

其他命令

felint checkrc

felint checkrc命令可以帮你列出当前路径及其父路径上存在的.eslintrc文件。

felint 依赖

felint 依赖包如下：

scss_lint@0.48.0

eslint@2.11.1

babel-eslint@6.0.4

eslint-plugin-react@5.1.1

eslint-plugin-jsx-a11y@1.2.3

eslint-config-airbnb@9.0.1

eslint-plugin-lean-imports@0.3.3

依赖将在执行felint init命令是自动安装。

Q&A

如发现git Hooks执行报错，请检查依赖版本是否符合。
